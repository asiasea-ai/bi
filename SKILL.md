# 金灯塔胜算 Skill (OpenClaw 简化版)

## 描述

本 skill 是金灯塔胜算业务系统的 OpenClaw 简化版本。基于 LLM 的意图识别与参数提取，支持用户通过完全无预设的**自然语言**来管理会话状态并查询 BI 数据报表。
本插件采用无头工具（Headless Tool）架构，剥离了传统的固定格式指令，实现了真正的多用户并发状态隔离。

---

## 核心能力

本插件向大模型暴露以下数据接口：
1. **系统初始化 (`initialize`)** - 加载初始权限并默认切入基础环境。
2. **系统检索 (`list_systems`)** - 获取系统内支持的所有子业务模块。
3. **环境切换 (`switch_system`)** - 调整当前会话处于哪个子系统环境（如：销售系统、财务系统）。
4. **BI 数据拉取 (`query_bi_data`)** - 接收 LLM 解析出的确切指标（metric）与时间范围（time_range），返回原生数据结构，交由 LLM 进行可视化或总结输出。

---

## 用户交互示例（无需固定指令）

用户无需记忆任何斜杠（`/`）命令，直接以口语化的方式与大模型沟通即可。大模型将自动调度本插件的底层工具：

- **初始化/鉴权**
  - “帮我初始化一下金灯塔数据系统。”
  - “我第一次用，怎么接入胜算平台？”

- **系统切换**
  - “你们现在支持查哪些系统的数据？”
  - “帮我把环境切到财务系统。”

- **数据查询**
  - “给我看下上个月各个渠道的销售额情况。”
  - “帮我拉一下最近7天的订单量数据。”

---

## 技术架构特点

1. **状态沙箱隔离**：抛弃全局变量，根据 `user_id` 在本地动态生成 `.session_{user_id}.json`，支持多用户并发调用不串号。
2. **防崩溃处理**：全面接管 Argparse 与 JSON 解析的异常层，无论遇到何种传参错误，均向引擎输出标准 JSON 错误日志。
3. **强制参数提取**：要求 LLM 必须提取时间（`time_range`），禁止默认静默查询，培养大模型的主动反问能力。
